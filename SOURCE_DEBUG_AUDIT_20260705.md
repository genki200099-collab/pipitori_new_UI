## 修正内容

### ゲーム進行
- ピック候補が何らかの理由で0枚になった場合、`doPick()` が何もせず戻って進行停止する可能性を修正。
  - 候補0枚時はログを残してピックをスキップし、次のリードへ進むフェイルセーフを追加。
- `submitPickTargets()` でも候補0枚の異常系を安全に処理するよう修正。
- ラウンド終了時に `room.trick` をクリアするよう修正。
  - トリック終了直後にラウンド終了した場合も、場札と出札アイコン吹き出しが残り続けないようにした。
- 新規ゲーム開始時に `shootEvent`、`lastPickReveal`、`nextLead` などの一時状態をリセット。
- プレイヤー切断時、ホスト移譲後に `ensureProgress()` を呼び、CPU手番やCPUピックが止まりにくいようにした。

### UI
- テーブル下部の `SHOW` ボタンを、機能が分かりやすい `ログ` に変更。
- 開始時ペア捨て画面を改善。
  - ペアにできるカードだけを表示。
  - ペア候補がない場合は説明を出し、スキップ操作だけを促す。
  - 異なる数字2枚を選べてしまう分かりづらさを解消。
- ペア浄化チャンスのスキップボタンを `浄化しない` に変更。
- UI用に `emptyHint`、`disabledCard` スタイルを追加。

## 追加全体点検 v5（2026-07-05）

### 点検対象
- `server.js` のトリック結果 → ピック → 次トリック進行
- `public/index.html` のピック待機・ペア浄化待機・選択状態リセット
- スマホ/PC共通で見える進行メッセージ

### 修正内容
1. **非表示ピック結果（finishingPick）から進まない可能性の補強**
   - `schedulePickFinish()` を堅牢化し、`resultAt` 欠損時の補正、タイマー `unref()`、安全な delay 範囲を追加。
   - `watchdogRooms()` と `ensureProgress()` に、`hideResult` 用の短い進行期限を追加。
   - タイマーが消失しても `finishAfterPick()` へ進むようにした。

2. **ピック用タイマーの残留対策**
   - `clearPickTimers()` を追加。
   - 新しいピック開始時とピック終了時に、古い `pickFinish / cpuPick / cpuPair / delayedPick` をクリア。
   - 前のピックの遅延処理が次のピックへ干渉しにくくした。

3. **次リード決定の安全化**
   - `finishAfterPick()` で winnerPid が不正になった場合のフォールバックを追加。
   - メッセージ生成時も存在しないプレイヤー参照で落ちないようにした。

4. **UIの待機説明を改善**
   - 他プレイヤーがペア浄化を選んでいる時、単に「ピックしています」ではなく「ペア浄化するか選んでいます」と表示。
   - ペア浄化待機中は引かれたカードも表示するように変更。

5. **選択状態の持ち越し対策を強化**
   - `interactionKey` に pendingPick の勝者/最弱/readyAt を含め、同じモード名の連続ピックでも古い選択が残りにくいようにした。

6. **人間プレイヤーアイコンの重複対策**
   - 1人目は 🐷 固定、2人目以降のランダム候補から 🐷 を除外。
