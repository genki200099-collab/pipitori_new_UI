# Mobile UI Optimization Audit

## 対応内容

- スマホ縦向き表示でゲーム卓が右側へはみ出す問題を修正。
  - 900px以下の縦向きでは、プレイヤー一覧を2列グリッド化。
  - 卓エリアをプレイヤー一覧の下に全幅配置。
  - 手札ドックとアクションボタンの安全領域を確保。

- スマホ横向き表示で手札・SHOWボタン・場札が重なる問題を修正。
  - 低高さの横向き用レイアウトを追加。
  - 卓エリア下部に手札専用の余白を確保。
  - 手札カード、場札カード、スコア欄をコンパクト化。

- 手札カードの重なりを動的調整。
  - 画面幅と手札枚数からカード間マージンを計算。
  - 13枚時でもスマホ幅に収まりやすく調整。
  - 回転角をモバイルでは抑え、見切れを軽減。

- PC表示への影響を抑制。
  - 通常のPC幅・高さでは従来レイアウトを維持。
  - 追加CSSは主に `orientation:portrait` / 低高さ横向き / 小幅条件でのみ発動。

## 検証

- `server.js` の構文チェック OK。
- `public/index.html` 内 JavaScript の構文チェック OK。

## 2026-07-05 UIチラつき監査 v6
- スマホ縦向き・横向き・PC表示で再描画処理を点検。
- プレイヤー欄、ターン順、場札、ログをキー比較して、内容が変わらない時はDOMを作り直さないよう修正。
- CPUセリフの pop アニメーションが状態更新のたびに再発火してチラつく問題を抑制。
- 場札カードの drop アニメーションが同じ場札のまま再実行される問題を抑制。
- レイアウト実測値CSS変数の更新を差分更新 + requestAnimationFrame 化し、スマホでの細かい上下揺れを軽減。
- 公開情報モーダル表示中に背後のピックモーダルが再表示される可能性を抑制。


## 2026-07-05 PC/スマホ整合レイアウト追加修正
- PCブラウザで出札アイコン吹き出しがカードの上に重なり、カード内容が読めなくなる問題を修正。
- PCでは出札アイコンをカードの外側へ逃がし、カード自体を上レイヤーにして視認性を優先。
- 左側プレイヤー欄のCPUセリフを絶対配置から行内表示へ変更し、コメント同士の重なりとスクロール発生を抑制。
- スマホ縦向き・横向きの既存レイアウトは維持し、PC向けの上書きは `@media (min-width:901px)` に限定。


## 2026-07-05 出札アイコン三角形削除
- PC/スマホ共通で、出札アイコン横の吹き出し三角形が崩れて見える問題に対応。
- アイコン本体とラベル表示は維持し、`.playAvatarTail` を非表示化。
- `showPlayedAvatar()` の生成HTMLからも三角形要素を削除し、表示崩れの原因をなくした。


## 2026-07-05 全体デバッグ点検 v7
- 切断中の人間プレイヤーが手番・3枚パス・開始時ペア捨て・ピック候補選択・公開ピック・ペア浄化待ちに入った場合、ゲーム進行が止まる可能性を修正。
- 切断中の人間プレイヤーは一定時間後にCPU相当の自動処理へフォールバックするようにした。
- ピックやシュート演出がサーバー側で期限切れになった後、クライアント上で古いオーバーレイが残る可能性を修正。
- ラウンド継続時に `lastPickReveal` / `shootEvent` を明示的にクリアし、次ラウンドへ古い演出が残らないようにした。


## 2026-07-05 UI表示安定化 v8
- スマホ縦向き・スマホ横向き・PCブラウザ表示で、表示の揺れやチラつきが起きやすい箇所を追加点検。
- CPUコメント欄を常時確保するようにし、コメントが出たり消えたりした時にプレイヤー欄の高さが跳ねる問題を軽減。
- `resize` / `orientationchange` 時の再描画を即時連打ではなく短時間デバウンスし、スマホ回転時のちらつきを抑制。
- 画面回転時は古い出札アイコン位置がズレて残らないよう、出札アイコンをいったんクリア。
- `prefers-reduced-motion` 環境で出札アイコンの消去アニメーションが発火せず、アイコンが残る可能性をフォールバックタイマーで修正。
- `scoreBoard` / `tableArea` / `handDock` にレイアウト安定化用の containment と scrollbar-gutter を追加。


## 2026-07-05 横向きスマホの場札配置修正 v9
- スマホ横向きで中央の場札カードが重なり、数字が読めなくなる問題を修正。
- 横向きスマホでは `trickZone` の最低高さを確保し、場札カードを少し小さくして四方向へ逃がす配置に変更。
- 高さ430px以下の端末向けにさらにコンパクトな場札サイズを追加。
- PC表示とスマホ縦向きの既存レイアウトには影響しないよう、`orientation:landscape` かつ低高さ画面向けのメディアクエリに限定。
