# Mobile UI Optimization Audit

## 対応内容

- スマホ縦向き表示でゲーム卓が右側へはみ出す問題を修正。
  - 900px以下の縦向きでは、プレイヤー一覧を2列グリッド化。
  - 卓エリアをプレイヤー一覧の下に全幅配置。
  - 手札ドックとアクションボタンの安全領域を確保。

- スマホ横向き表示で手札・SHOWボタン・場札が重なる問題を修正。
  - 低高さの横向き用レイアウトを追加。
  - 卓エリア下部に手札専用の余白を確保。
  - 手札カード、場札カード、スコア欄をコンパクト化。

- 手札カードの重なりを動的調整。
  - 画面幅と手札枚数からカード間マージンを計算。
  - 13枚時でもスマホ幅に収まりやすく調整。
  - 回転角をモバイルでは抑え、見切れを軽減。

- PC表示への影響を抑制。
  - 通常のPC幅・高さでは従来レイアウトを維持。
  - 追加CSSは主に `orientation:portrait` / 低高さ横向き / 小幅条件でのみ発動。

## 検証

- `server.js` の構文チェック OK。
- `public/index.html` 内 JavaScript の構文チェック OK。
