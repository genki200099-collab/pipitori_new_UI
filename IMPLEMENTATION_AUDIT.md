# ピピトリ 修正・点検メモ

## 修正日
2026-07-04

## 対応した修正

- 再接続対応を追加
  - ブラウザの `localStorage` に部屋コード、プレイヤーID、セッショントークンを保存。
  - リロードや通信断後、同じ席へ `resumeRoom` で復帰。
  - 満員部屋でも正規セッションなら復帰可能。
- ホスト切断時のホスト移譲を追加
  - 接続中の人間プレイヤーがいる場合、そのプレイヤーへホスト権限を移譲。
- 次ラウンド補充時の抜き札復活を修正
  - 初期配札で抜いた通常カードを `room.removedCard` として保持。
  - `refillHands()` でもそのカード面を補充候補から除外。
- タイマー管理を強化
  - `review` / `pickFinish` / `cpu` / `cpuPick` / `cpuPair` を一括クリアする `clearGameplayTimers()` を追加。
  - ラウンド終了、次ラウンド開始、ゲーム開始で古いタイマーを除去。
  - CPUピック・CPUペア処理に token 照合を追加し、古いタイマーが新しい処理へ干渉しないようにした。
- ロビー表示のHTMLエスケープ
  - プレイヤー名と画像パスをエスケープし、表示崩れ・HTML混入を抑止。
- スマホ横画面の高さ不足対策
  - `max-height:520px` 用の圧縮レイアウトを追加。
  - 下部手札、スコア欄、場札、モーダルを低い画面でも見えるよう縮小。
- ちらつき軽減
  - 手札UIとピックUIに描画キーを追加し、状態が変わっていない時は再描画をスキップ。
  - 公開ピック演出中はピック画面を一時的に隠し、演出終了後に必要なら再表示。
  - 公開ピック演出・シュート演出の終了タイマーにID照合を追加し、古い演出タイマーが新しい演出を消さないようにした。

## 2026-07-05 追加全体デバッグ点検 v3
- 公開ピックで、準備時間中にクリックするとサーバー側で無視され、ユーザー視点では止まったように見える問題を修正。
  - サーバー側：早すぎる doPick は `delayedPick` タイマーで準備完了後に実行するフェイルセーフを追加。
  - フロント側：準備中の裏向きカードを一時的に disabled 表示にして、準備完了後に自動で再描画。
- フェーズ切り替え時に古いカード選択状態が残り、別画面で選択済み表示が残る可能性を修正。
- 進行系タイマーのクリア対象に `delayedPick` を追加。
- `disabledCard` を pointer-events 無効化し、押せない状態が見た目と挙動で一致するよう調整。

## 2026-07-05 追加全体点検 v4
- 進行停止対策として、サーバー側に1秒間隔の軽量 watchdog を追加。
  - トリック結果確認タイマーが何らかの理由で進まなかった場合、`trickReview.until` 経過後に自動でピック処理へ進めます。
  - ピック結果表示後に `pickFinish` タイマーが失われた場合も、一定時間後に次リードへ進めます。
- 公開ピックの準備時間中にユーザーがカードを押した場合、クリックを無視して止まったように見える問題を修正。
  - サーバー側は準備完了後に delayedPick として実行。
  - フロント側は準備中カードを disabled 表示し、準備完了時に自動再描画します。
- フェーズ切り替え時に古い選択状態が残る可能性を修正。
- 静的ファイル配信で不正なURLエンコードが来てもサーバーが落ちないよう、`decodeURIComponent` を try/catch 化。
- `server.js` と `public/index.html` 内 JavaScript の構文チェックを実施。

- 2026-07-05: ピック公開情報とピック結果が重複するUIを整理。ペア浄化候補がない場合は結果モーダルを省略し、公開情報だけを表示して次のトリックへ進むようにした。

## 2026-07-05 全体デバッグ点検 v7

## 2026-07-05 UI表示安定化 v8
- スマホ縦向き・スマホ横向き・PCブラウザ表示で、表示の揺れやチラつきが起きやすい箇所を追加点検。
- CPUコメント欄を常時確保するようにし、コメントが出たり消えたりした時にプレイヤー欄の高さが跳ねる問題を軽減。
