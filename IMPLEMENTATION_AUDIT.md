# ピピトリ 修正・点検メモ

## 修正日
2026-07-04

## 対応した修正

- 再接続対応を追加
  - ブラウザの `localStorage` に部屋コード、プレイヤーID、セッショントークンを保存。
  - リロードや通信断後、同じ席へ `resumeRoom` で復帰。
  - 満員部屋でも正規セッションなら復帰可能。
- ホスト切断時のホスト移譲を追加
  - 接続中の人間プレイヤーがいる場合、そのプレイヤーへホスト権限を移譲。
- 次ラウンド補充時の抜き札復活を修正
  - 初期配札で抜いた通常カードを `room.removedCard` として保持。
  - `refillHands()` でもそのカード面を補充候補から除外。
- タイマー管理を強化
  - `review` / `pickFinish` / `cpu` / `cpuPick` / `cpuPair` を一括クリアする `clearGameplayTimers()` を追加。
  - ラウンド終了、次ラウンド開始、ゲーム開始で古いタイマーを除去。
  - CPUピック・CPUペア処理に token 照合を追加し、古いタイマーが新しい処理へ干渉しないようにした。
- ロビー表示のHTMLエスケープ
  - プレイヤー名と画像パスをエスケープし、表示崩れ・HTML混入を抑止。
- スマホ横画面の高さ不足対策
  - `max-height:520px` 用の圧縮レイアウトを追加。
  - 下部手札、スコア欄、場札、モーダルを低い画面でも見えるよう縮小。
- ちらつき軽減
  - 手札UIとピックUIに描画キーを追加し、状態が変わっていない時は再描画をスキップ。
  - 公開ピック演出中はピック画面を一時的に隠し、演出終了後に必要なら再表示。
  - 公開ピック演出・シュート演出の終了タイマーにID照合を追加し、古い演出タイマーが新しい演出を消さないようにした。

## 残る注意点

- 実機スマホでの横画面表示は、端末ごとのブラウザUI高さ差があるため、Render公開後にiPhone / Androidで最終確認推奨。
- WebSocket通信・再接続はローカルで簡易確認済みだが、Renderのスリープ復帰・ネットワーク揺れでは追加確認推奨。
